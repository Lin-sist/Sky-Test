# Sky Tracker 作战系统（独立编码 + 排错优先）

## 你的目标（当前版本）
- 主目标：提升独立编码能力、排错能力
- 次目标：沉淀可迁移到 Spring Boot RAG 项目的通用能力
- 节奏约束：每天 1 小时，短周期冲刺，不做长期大而全

## 使用原则（很重要）
1. 每次开发前，先写“任务卡”的伪代码与边界条件。
2. 每次报错后，必须写 RCA（根因复盘），不允许只记录“已修复”。
3. 每完成一个功能，必须提炼“可迁移模式”，否则视为未完成。
4. 每周做一次复盘，决定“保留什么能力、砍掉什么任务”。

## 目录说明
- 01-项目追踪：阶段状态与当前任务入口
- 02-任务卡：功能级执行卡（设计→编码→验证→复盘）
- 03-RCA故障复盘：错误定位与证据链
- 04-可迁移模式库：从业务代码抽象工程能力
- 05-知识盲区：概念理解分级与通关证据
- 06-周复盘：每周投入产出与计划纠偏
- 07-30天冲刺：1小时/天的落地路线

## 每天 1 小时最小闭环（建议）
- 10 分钟：读任务卡 + 明确验收标准
- 30 分钟：独立编码（不看参考源码）
- 15 分钟：跑验证/制造破坏性测试
- 5 分钟：更新 RCA 或模式库（至少一条）

## 完成定义（DoD）
一个功能点只有同时满足以下条件才算“完成”：
- 代码可运行
- 任务卡已填完（含边界条件）
- 至少 1 条破坏性测试结果
- 至少 1 条可迁移模式沉淀
